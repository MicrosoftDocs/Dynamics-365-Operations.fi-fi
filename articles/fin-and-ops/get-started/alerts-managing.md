### <a name="frequency-for-change-based-events"></a><span data-ttu-id="2876d-116">Muutoksiin perustuvien tapahtumien aikaväli</span><span class="sxs-lookup"><span data-stu-id="2876d-116">Frequency for change-based events</span></span>
<span data-ttu-id="2876d-117">Voit määrittää muutoksiin perustuvia tapahtumia varten eräkäsittelytehtävän, joka käynnistää tapahtuman käsittelemisen nopeasti tapahtuman järjestelmään kirjaamisen jälkeen.</span><span class="sxs-lookup"><span data-stu-id="2876d-117">For change-based events, you can set up a batch job that triggers the processing of an event soon after the event is logged by the system.</span></span> <span data-ttu-id="2876d-118">Jos määrität erätyön useammin tapahtuvaksi, käyttäjät saavat hälytykset nopeammin muutosten tekemisen jälkeen.</span><span class="sxs-lookup"><span data-stu-id="2876d-118">If you set up the batch job to recur more often, users receive their alerts sooner after changes occur.</span></span> <span data-ttu-id="2876d-119">Usein suoritettava eräkäsittely voi kuitenkin vaikuttaa järjestelmän suorituskykyyn.</span><span class="sxs-lookup"><span data-stu-id="2876d-119">However, a high frequency for batch processing might adversely affect system performance.</span></span>

<span data-ttu-id="2876d-120">Toisaalta erätyö, jota ei suoriteta niin usein, ja joka ajoitetaan aikaan, jolloin järjestelmän kuormitus on alhainen, voi parantaa järjestelmän suorituskykyä.</span><span class="sxs-lookup"><span data-stu-id="2876d-120">On the other hand, a batch job that recurs less often, and that is scheduled for times when the system load is low, might help improve system performance.</span></span> <span data-ttu-id="2876d-121">Harvoin suoritettava eräkäsittely ei kuitenkaan ehkä täytä käyttäjien hälytysvaatimuksia.</span><span class="sxs-lookup"><span data-stu-id="2876d-121">However, a low frequency for batch processing might not meet the users' demands for timely alerts.</span></span>

<span data-ttu-id="2876d-122">Kun määrität muutoksiin perustuvien tapahtumien eräkäsittelyn aikavälejä, ota tämän vuoksi huomioon hälytysten aikarajat ja koko järjestelmän suorituskyky.</span><span class="sxs-lookup"><span data-stu-id="2876d-122">Therefore, when you set up the frequency of batch processing for change-based events, consider the balance between the timeliness of alerts and the performance of the whole system.</span></span> <span data-ttu-id="2876d-123">Tämä on erityisen tärkeää silloin, kun hälytyssääntöjä luovien käyttäjien määrä lisääntyy.</span><span class="sxs-lookup"><span data-stu-id="2876d-123">These considerations become more relevant as the number of users who create alert rules increases.</span></span> <span data-ttu-id="2876d-124">Tiheys ei vaikuta käsiteltävien tapahtumien määrään.</span><span class="sxs-lookup"><span data-stu-id="2876d-124">The frequency doesn't affect the number of events that must be processed.</span></span> <span data-ttu-id="2876d-125">Jos useammat käyttäjät luovat sääntöjä, on suoritettava useampia tarkistuksia.</span><span class="sxs-lookup"><span data-stu-id="2876d-125">However, if more users create rules, more checks must be done.</span></span> <span data-ttu-id="2876d-126">Tietojen tämän tyyppinen vaihtaminen voi vaikuttaa järjestelmän suorituskykyyn.</span><span class="sxs-lookup"><span data-stu-id="2876d-126">This type of data exchange might affect system performance.</span></span>

#### <a name="the-risks-of-low-batch-frequency"></a><span data-ttu-id="2876d-127">Pitkillä aikaväleillä toteutettujen eräkäsittelyjen riskit</span><span class="sxs-lookup"><span data-stu-id="2876d-127">The risks of low batch frequency</span></span>
<span data-ttu-id="2876d-128">Jos määrität muutosperustaisten tapahtumien eräkäsittelyn aikavälit pitkiksi, hälytyssääntöjen ehtojen tiedot voivat muuttua ennen erän käsittelyä.</span><span class="sxs-lookup"><span data-stu-id="2876d-128">If you set up a low frequency for batch processing for change-based events, data that is relevant to the conditions in alert rules might be changed before the batch is processed.</span></span> <span data-ttu-id="2876d-129">Voit menettää hälytykset tämän vuoksi.</span><span class="sxs-lookup"><span data-stu-id="2876d-129">Therefore, you might lose alerts.</span></span>

<span data-ttu-id="2876d-130">Esimerkiksi hälytyssääntö on määritetty käynnistämään hälytys, kun tapahtuma on **asiakkaan yhteystietojen muutokset** ja ehtona on **asiakas = BB**.</span><span class="sxs-lookup"><span data-stu-id="2876d-130">For example, an alert rule is set up to trigger an alert when the event is **customer contact changes** and the condition is **customer = BB**.</span></span> <span data-ttu-id="2876d-131">Toisin sanoen tapahtuma kirjataan, kun asiakkaan BB yhteyshenkilöä muutetaan.</span><span class="sxs-lookup"><span data-stu-id="2876d-131">In other words, when the customer contact for customer BB is changed, the event is logged.</span></span> <span data-ttu-id="2876d-132">Eräkäsittelyjärjestelmä määritetään kuitenkin niin, että eräkäsittely tapahtuu tietojen merkintää harvemmin.</span><span class="sxs-lookup"><span data-stu-id="2876d-132">However, the batch processing system is set up so that batch processing occurs less often than data entry.</span></span> <span data-ttu-id="2876d-133">Jos asiakkaan nimi muuttuu nimestä **BB** nimeksi **AA** ennen uuden tapahtuman käsittelyä, tietokannan tiedot eivät enää vastaa säännön ehtoa **asiakas = BB**.</span><span class="sxs-lookup"><span data-stu-id="2876d-133">If the customer name is changed from **BB** to **AA** before the event is processed, the data in the database no longer matches the condition in the rule, **customer = BB**.</span></span> <span data-ttu-id="2876d-134">Tämän vuoksi hälytystä ei muodosteta, kun tapahtuma lopulta käsitellään.</span><span class="sxs-lookup"><span data-stu-id="2876d-134">Therefore, when the event is finally processed, no alert is generated.</span></span>
