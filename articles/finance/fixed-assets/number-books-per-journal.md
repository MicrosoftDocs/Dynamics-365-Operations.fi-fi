[!include [banner](../includes/banner.md)]

<span data-ttu-id="87f03-105">Tässä ohjeaiheessa on tietoja kirjauskansioiden ja resurssin kirjojen välisistä suhteista, kun käyttöomaisuushankinta tai poistoehdotus tehdään erätyön kautta.</span><span class="sxs-lookup"><span data-stu-id="87f03-105">This topic describes the relationship between journals and asset books when you create a fixed asset acquisition or depreciation proposal through a batch job.</span></span> <span data-ttu-id="87f03-106">Voit määrittää kuhunkin hankintaan ja poistoon sisällytettävien kirjojen enimmäismäärän käyttämällä **Kirjojen määrä kirjauskansion mukaan** -osan **Yleistiedot**-välilehden **Käyttöomaisuusparametrit**-sivun kenttiä (**Käyttöomaisuus \> Asetukset \> Käyttöomaisuusparametrit**).</span><span class="sxs-lookup"><span data-stu-id="87f03-106">You can define the maximum number of books that are included for each acquisition and for depreciation by using the fields in the **Number of books per journal** section on the **General** tab of the **Fixed assets parameters** page (**Fixed assets \> Setup \> Fixed assets parameters**).</span></span> <span data-ttu-id="87f03-107">Näiden kenttien avulla voit jakaa käyttöomaisuuskirjojen määrän hankintakirjauskansiota ja poistokirjauskansiota kohden.</span><span class="sxs-lookup"><span data-stu-id="87f03-107">These fields let you distribute the number of asset books per acquisition journal and depreciation journal.</span></span>

<span data-ttu-id="87f03-108">Hankintaehdotuksen oletusarvo on vähintään 10 000 kirjaa.</span><span class="sxs-lookup"><span data-stu-id="87f03-108">For an acquisition proposal, the default value is at least 10,000 books.</span></span> <span data-ttu-id="87f03-109">Poistoehdotuksen oletusarvo on vähintään 2 000 kirjaa.</span><span class="sxs-lookup"><span data-stu-id="87f03-109">For a depreciation proposal, the default value is at least 2,000 books.</span></span>

<span data-ttu-id="87f03-110">Jos käyttöomaisuuseriä on esimerkiksi 4 000, kuhunkin erään liitetään kaksi kirjaa. Yksi kirja kirjataan nykyiselle tasolle ja toinen verotasolle.</span><span class="sxs-lookup"><span data-stu-id="87f03-110">For example, if there are 4,000 fixed assets, and two books are associated with each asset, one book will be posted to the current layer, and the other book will be posted to the tax layer.</span></span> <span data-ttu-id="87f03-111">Jos hankit 4 000 käyttöomaisuuserää eräkäsittelyn avulla, erätyö, joka luo yhden käyttöomaisuuserän hankintakirjauskansion, sisältää 4 000 käyttöomaisuuskirjaa.</span><span class="sxs-lookup"><span data-stu-id="87f03-111">If you acquire 4,000 fixed assets through batch processing, the batch job that creates one fixed asset acquisition journal will contain 4,000 asset books.</span></span>

> [!NOTE]
> <span data-ttu-id="87f03-112">Luotua kirjauskansiota käytetään edelleen, kunnes erätyö on päättynyt.</span><span class="sxs-lookup"><span data-stu-id="87f03-112">The journal that is created will continue to be used until the batch job is completed.</span></span>
>
> <span data-ttu-id="87f03-113">Johdetut kirjat eivät sisälly kirjauskansion kirjauskansiokohtaiseen enimmäismäärään.</span><span class="sxs-lookup"><span data-stu-id="87f03-113">The derived books aren't included in the maximum number of books per journal.</span></span>

<span data-ttu-id="87f03-118">Eräkäsittelytyö sulkee pois suljetut kirjat.</span><span class="sxs-lookup"><span data-stu-id="87f03-118">The batch processing job excludes closed books.</span></span> <span data-ttu-id="87f03-119">Esimerkiksi poistojen eräajossa 10 ensimmäistä kirjaa 2 000 kirjasta on suljettu.</span><span class="sxs-lookup"><span data-stu-id="87f03-119">For example, in a batch job for depreciation, 10 of the first 2,000 books are closed.</span></span> <span data-ttu-id="87f03-120">Tässä tapauksessa ensimmäinen kirjauskansio sisältää kirjoja, jotka on liitetty käyttöomaisuuseriksi, joiden numero on 1-2 011.</span><span class="sxs-lookup"><span data-stu-id="87f03-120">In this case, the first journal will contain books that are associated with the fixed assets that are numbered 1 through 2,011.</span></span> <span data-ttu-id="87f03-121">Toinen kirjauskansio sisältää kirjoja, jotka on liitetty käyttöomaisuuseriksi, joiden numero on 2 012–4 000.</span><span class="sxs-lookup"><span data-stu-id="87f03-121">The second journal will then contain books that are associated with the fixed assets that are numbered 2,012 through 4,000.</span></span>

> [!NOTE]
> <span data-ttu-id="87f03-122">Jos sinulla on käyttöomaisuustunnuksen, joilla on eri erottimet (kuten – tai /), ja luot käyttöomaisuustapahtumia eräajoissa, kullekin erotintyypille on tehtävä erillinen eräajo.</span><span class="sxs-lookup"><span data-stu-id="87f03-122">If you have fixed asset IDs with different separators (such as – or /), and you create fixed asset transactions in batch jobs, you must run a separate batch job for each type of separator.</span></span> <span data-ttu-id="87f03-123">Järjestelmä ei voi käsitellä eri erottimia samassa eräajossa.</span><span class="sxs-lookup"><span data-stu-id="87f03-123">The system can't process different separators within the same batch job.</span></span>

data-ttu-id="87f03-129">Tuloksena on 2 001 kirjaa.</span><span class="sxs-lookup"><span data-stu-id="87f03-129">The result will be 2,001 books.</span></span> <span data-ttu-id="87f03-130">Tämän vuoksi three months, there will be 6,003 journal lines to maintain the same asset IDs in the same journal.</span></span> <span data-ttu-id="87f03-131">Järjestelmä luo myös yhden kirjauskansion, jossa on 332 asset IDs and three books for each fixed asset ID.</span></span> <span data-ttu-id="87f03-132">Kolmessa kuukaudessa saadaan 2 988 riviä.</span><span class="sxs-lookup"><span data-stu-id="87f03-132">In > [!NOTE] 
> <span data-ttu-id="87f03-133">Jos **Tee yhteenveto poistoista** -parametri on käytössä poistoehdotusta luotaessa, **Kirjojen määrä kirjauskansiota kohti – poistoehtotus** -kentän arvolla ei ole vaikutusta.</span><span class="sxs-lookup"><span data-stu-id="87f03-133">If the **Summarize depreciation** parameter is turned on when you're creating a depreciation proposal, then the value in the **Number of books per journal - Depreciation proposal** field has no effect.</span></span> <span data-ttu-id="87f03-134">Tässä tapauksessa kirjauskansiokohtainen kirjamäärä on 6 000 eli sisäinen määritetty raja.</span><span class="sxs-lookup"><span data-stu-id="87f03-134">In this case number of books per journal is 6000, which is the internal defined limit.</span></span>
