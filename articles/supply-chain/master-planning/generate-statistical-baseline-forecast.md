[!include [banner](../includes/banner.md)]

<span data-ttu-id="5a3f4-104">Tässä ohjeaiheessa on tietoja kysynnän ennusteissa käytetyistä parametreista ja suodattimista.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-104">This topic provides information about the parameters and filters that are used in the calculation of demand forecasting.</span></span> 

<span data-ttu-id="5a3f4-105">Kun luot perusennusteen, määritä ensin laskennassa käytettävät parametrit ja suodattimet.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-105">When you create a baseline forecast, you must first specify the parameters and filters that are used in the calculation.</span></span> <span data-ttu-id="5a3f4-106">Voit esimerkiksi luoda perusennusteen, joka arvioi kysyntää tietyn yrityksen edellisen vuoden tapahtumatietojen perusteella tulevalle kuukaudelle ja valitulle nimikeryhmälle.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-106">For example, you can create a baseline forecast that estimates demand based on transaction data from the past year for a specific company, for the coming month, and for a selected group of items.</span></span> 

<span data-ttu-id="5a3f4-107">Luo kysynnän ennuste kohdassa **Pääsuunnittelu &gt; Ennuste &gt; Kysynnän ennuste &gt; Luo tilastollinen perusennuste**.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-107">To generate a demand forecast, go to **Master planning &gt; Forecasting &gt; Demand forecasting &gt; Generate statistical baseline forecast**.</span></span> 

<span data-ttu-id="5a3f4-108">Ennustejakso voidaan valita ennustetta luotaessa.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-108">The forecast bucket can be selected at forecast generation time.</span></span> <span data-ttu-id="5a3f4-109">Käytettävissä olevat arvot ovat: Päivä, Viikko ja Kuukausi.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-109">The available values are: Day, Week, and Month.</span></span> 

<span data-ttu-id="5a3f4-110">Aikajaksojen lukumäärä, joille ennuste luodaan, määritetään **Ennustehorisontti**-kentässä.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-110">The number of buckets to generate a forecast for is set in the **Forecast horizon** field.</span></span> 

<span data-ttu-id="5a3f4-111">Kun ennustestrategiaksi on määritetty **Kopioi historiallisen kysynnän päälle**, historiallisen horisontin päättymispäivä jätetään huomiotta.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-111">When the forecast strategy is set to **Copy over historical demand**, the end of the historical horizon is ignored.</span></span> <span data-ttu-id="5a3f4-112">Järjestelmä kopioi **Ennustehorisontti**-kentässä määritettyjen aikajaksojen lukumäärän kysynnän ennusteeseen, alkaen **Päivämäärästä**-kentässä **Historiallinen horisontti** -kohdassa määritetystä päivämäärästä.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-112">The system copies the number of buckets specified in the **Forecast horizon** field to the forecast demand, starting from the date set in the **From date** field under **Historical horizon**.</span></span> <span data-ttu-id="5a3f4-113">Kopioimalla historiallisen kysynnän määrätystä päivämäärästä eteenpäin, tuotannon suunnittelijat voivat tehdä suunnitelman seuraavalle neljännesvuodelle kahdella tavalla:</span><span class="sxs-lookup"><span data-stu-id="5a3f4-113">By copying historical demand from a certain date forward, production planners can make the plan for the next quarter in two ways:</span></span>

-   <span data-ttu-id="5a3f4-114">Kopioimalla kysynnän samasta neljännesvuodesta viime vuonna.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-114">By copying the demand from the same quarter last year.</span></span>
-   <span data-ttu-id="5a3f4-115">Kopioimalla kysynnän edellisestä neljännesvuodesta.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-115">By copying the demand from the previous quarter.</span></span>

<span data-ttu-id="5a3f4-116">Tuotantosuunnitelmiin aiheutuvien sekaannusten välttämiseksi on mahdollista lukita määrätty ennustejaksojen määrä.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-116">To prevent confusion in the production plans, a certain number of forecast buckets can be frozen.</span></span> <span data-ttu-id="5a3f4-117">Tämä luku määritetään **Lukitusaikaraja**-kentässä.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-117">This number is set in the **Freeze time fence** field.</span></span> <span data-ttu-id="5a3f4-118">Sivulla **Oikaistu kysynnän ennuste**, lukittujen aikajaksojen kentät on poistettu käytöstä. Näin annetaan visuaalinen osoitus, että näitä arvoja ei tule muuttaa.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-118">On the **Adjusted demand forecast** page, the cells for the frozen buckets are disabled, to give a visual indication that those values should not be changed.</span></span> 

<span data-ttu-id="5a3f4-119">Perusmuotoisen kysynnän ennusteen alkamispäivämäärän ei tarvitse olla nykyinen päivä tai tuleva päivä.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-119">The start date for the baseline demand forecast doesn’t have to be the current date or a date in the future.</span></span> <span data-ttu-id="5a3f4-120">Määritä uusi alkupäivämäärä **Perusennusteen alkamispäivä - alkaen** -kentässä.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-120">To set a different start date, use the **Baseline forecast start date - From date** field.</span></span> <span data-ttu-id="5a3f4-121">Esimerkiksi kesäkuussa käyttäjät voivat luoda ennusteen seuraavalle vuodelle.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-121">For example, in June, users can generate a forecast for the next year.</span></span> <span data-ttu-id="5a3f4-122">Koska ennusteen aikajaksot historiallisen kysynnän päättymispäivän ja perusennusteen alkamispäivän välillä puuttuvat, ennusteet eivät ehkä ole oikeita.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-122">Because the forecast buckets between the end of historical demand and the start of the baseline are missing, the predictions might not be accurate.</span></span> <span data-ttu-id="5a3f4-123">Jos käytät kysynnän ennustepalvelua, voit täyttää puuttuvat kohdat neljällä tavalla.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-123">If you are using the Demand forecasting service, there are four ways in which you can fill in the missing gaps.</span></span> <span data-ttu-id="5a3f4-124">Voit valita haluamasi menetelmän määrittämällä MISSING\_VALUE\_SUBSTITUTION (Puuttuvan arvon korvaus) -parametrin **Kysynnän ennusteen parametrit** -sivulla.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-124">You can choose the method that you want by setting the MISSING\_VALUE\_SUBSTITUTION parameter on the **Demand forecasting parameters** page.</span></span> 

> [!NOTE]
> <span data-ttu-id="5a3f4-125">Puuttuvan arvon korvaus toimii vain historiallisten tietojen alkamis-ja päättymispäivämäärien välisten aukkojen tiedoissa.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-125">Missing value substitution only works for the gaps in data in between the start and end dates for historical data.</span></span> <span data-ttu-id="5a3f4-126">Se ei täytä tietoja ennen viimeisintä fyysistä tietopistettä eikä sen jälkeen, vaan se toimii olemassa olevien tietopisteiden välisenä ekstrapolaationa.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-126">It will not fill in data before or after the last physical data point, it only acts as extrapolation between actual existing data points.</span></span> 

<span data-ttu-id="5a3f4-127">**Perusennusteen alkamispäivä**  -  **Päivämäärästä** -kenttä on määritettävä ennusteaikajakson alkuun, esimerkiksi Yhdysvalloissa sunnuntaille, jos ennusteaikajakso on viikko.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-127">The **Baseline forecast start date** - **From date** field has to be set to the beginning of a forecast bucket, for example, in the United States, a Sunday if the forecasting bucket is the week.</span></span> <span data-ttu-id="5a3f4-128">Järjestelmä muokkaa automaattisesti **Perusennusteen alkamispäivä**  -  **Päivämäärästä** -kentän vastaamaan ennusteaikajakson alkua.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-128">The system automatically adjusts the **Baseline forecast start date** - **From date** field to match the beginning of a forecast bucket.</span></span> 

<span data-ttu-id="5a3f4-129">**Perusennusteen alkamispäivä**  -  **Päivämäärästä** -kenttään voidaan määrittää mennyt päivämäärä.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-129">The **Baseline forecast start date** - **From date** field can be set to a date in in the past.</span></span> <span data-ttu-id="5a3f4-130">On toisin sanoen mahdollista luoda kysynnän ennuste menneisyydelle.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-130">In other words, it is possible to generate a demand forecast in the past.</span></span> <span data-ttu-id="5a3f4-131">Tämä on hyödyllistä, koska sen avulla käyttäjät voivat optimoida ennustepalvelun parametrit siten, että tilastollinen menneisyydelle luotu ennuste vastaa todellista historiallista kysyntää.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-131">This is useful, because it lets users adjust the forecast service parameters so that the statistical forecast generated in the past matches the actual historical demand.</span></span> <span data-ttu-id="5a3f4-132">Käyttäjät voivat sitten jatkaa näiden parametriasetusten käyttöä luodakseen tilastollisen perusennusteen tulevaisuudelle.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-132">Users can then continue using these parameter settings to generate a statistical baseline forecast for the future.</span></span> 

<span data-ttu-id="5a3f4-133">Aiemmissa kysynnän ennusteen iteraatioissa tehtyjä manuaalisia oikaisuja voidaan käyttää automaattisesti uudessa perusennusteessa, jos **Siirrä manuaaliset oikaisut kysynnän ennusteeseen** -valintaruutu on valittuna.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-133">Manual adjustments made in previous demand forecasting iterations can be automatically applied to the new baseline forecast if the **Transfer manual adjustments to the demand forecas** t check box is selected.</span></span> <span data-ttu-id="5a3f4-134">Jos tämä valintaruutu on tyhjä, manuaalisia oikaisuja ei lisätä perusennusteeseen, mutta niitä ei poisteta.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-134">If the check box is cleared, the manual adjustments are not added to the baseline forecast – but they are not deleted.</span></span> <span data-ttu-id="5a3f4-135">Ennusteeseen tehdyt manuaaliset oikaisut voidaan poistaa vain ennusteen tuontihetkellä tyhjentämällä **Tallenna kysynnän perusennusteeseen tehdyt manuaaliset oikaisut** -valintaruutu.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-135">Manual adjustments made to a forecast can be deleted only at forecast import time, by clearing the **Save the manual adjustments made to the baseline demand forecast** check box.</span></span> <span data-ttu-id="5a3f4-136">Manuaaliset oikaisut tallennetaan varmennushetkellä.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-136">Manual adjustments are saved at authorization time.</span></span> <span data-ttu-id="5a3f4-137">Niinpä jos käyttäjä tekee manuaalisia oikaisuja ennusteeseen mutta ei varmenna ennustetta takaisin Supply Chain Managementiin, muutokset häviävät.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-137">Therefore, if a user makes manual adjustments to the forecast, but doesn’t authorize the forecast back to Supply Chain Management, the changes are lost.</span></span> <span data-ttu-id="5a3f4-138">Lisätietoja manuaalisista oikaisuista ja niiden toiminnasta on kohdassa [Oikaistun ennusteen varmennus](authorize-adjusted-forecast.md).</span><span class="sxs-lookup"><span data-stu-id="5a3f4-138">For more information about manual adjustments and how they work, see [Authorize an adjusted forecast](authorize-adjusted-forecast.md).</span></span> 

<span data-ttu-id="5a3f4-139">Kysynnän ennusteen luonnilla voi olla nimi ja kommentteja, jotka auttavat käyttäjiä tunnistamaan luodun ennusteen.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-139">A demand forecast generation can have a name and comments to help users identify the forecast that has been generated.</span></span> <span data-ttu-id="5a3f4-140">Nämä arvot näkyvät ennusteen luontihistoriassa **Tilastollisen perusennusteen luontihistoria** -sivulla.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-140">These values are visible in forecast generation history on the **Statistical baseline forecast generation history** page.</span></span> 

<span data-ttu-id="5a3f4-141">Konsernin sisäinen suunnitteluryhmä, nimikkeen kohdistustunnukset ja muut suodattimet voidaan määrittää ennusteen luontihetkellä.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-141">The intercompany planning group, item allocation keys, and other filters can be applied at forecast generation time.</span></span> <span data-ttu-id="5a3f4-142">Niitä voidaan käyttää parantamaan suorituskykyä tai jakamaan tiedot hallittaviin paloihin.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-142">These can be used to improve performance or to split the data into manageable chunks.</span></span> <span data-ttu-id="5a3f4-143">Huomaa kuitenkin, että kysynnän ennustetta ei luoda nimikkeen kohdistusryhmän jäsenille, joita ei ole määritetty konsernin sisäiseen suunnitteluryhmään, vaikka kyseinen nimikkeen kohdistustunnus olisi valittuna kyselyssä.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-143">However, note that a demand forecast is not generated for the members of any item allocation key that is not associated with an intercompany planning group, even if the item allocation key is selected in the query.</span></span> 

> [!TIP]
> <span data-ttu-id="5a3f4-144">Joskus käyttäjät saattavat saada virheitä luodessaan kysynnän ennusteita, tai kysynnän ennuste suoritetaan loppuun ilman istuntolokia.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-144">Sometimes users might receive errors while generating a demand forecast, or forecast generation is completed with no session log.</span></span> <span data-ttu-id="5a3f4-145">Tämän voivat aiheuttaa aiemmin ennusteen luonnissa käytetyt kyselyn ylijäämätiedot.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-145">This can happen because of leftover data in the query that was previously used for forecast generation.</span></span> <span data-ttu-id="5a3f4-146">Voit ratkaista tämän ongelman avaamalla **Kysely**-sivun valitsemalla **Valitse**, valitsemalla **Palauta** ja luomalla sitten perusennusteen.</span><span class="sxs-lookup"><span data-stu-id="5a3f4-146">To resolve this issue, click **Select** to open the **Query** page, select **Reset**, and then regenerate the baseline forecast.</span></span>
