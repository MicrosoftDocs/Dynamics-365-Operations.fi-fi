## <a name="work-templates"></a><span data-ttu-id="ca141-108">Työmallit</span><span class="sxs-lookup"><span data-stu-id="ca141-108">Work templates</span></span>

<span data-ttu-id="ca141-109">**Työmallit**-sivun avulla voit määrittää työtoiminnot, jotka on suoritettava varastossa.</span><span class="sxs-lookup"><span data-stu-id="ca141-109">The **Work templates** page lets you define the work operations that must be performed in the warehouse.</span></span> <span data-ttu-id="ca141-110">Yleensä varastotyötoiminnot kostuvat toimintopareista: varaston työntekijä keräilee käsillä olevan varaston yhdessä sijainnissa ja asettaa sitten keräillyt varastotuotteet toiseen paikkaan.</span><span class="sxs-lookup"><span data-stu-id="ca141-110">Typically, warehouse work operations consist of a pair of actions: a warehouse worker picks up on-hand inventory in one location and then puts the picked inventory down in another location.</span></span> 

<span data-ttu-id="ca141-111">Työmallit koostuvat otsikosta ja siihen liittyvistä riveistä.</span><span class="sxs-lookup"><span data-stu-id="ca141-111">Work templates consists of a header and associated lines.</span></span> <span data-ttu-id="ca141-112">Jokainen työmalli on tarkoitettu määrätylle *työtilaustyypille*.</span><span class="sxs-lookup"><span data-stu-id="ca141-112">Each work template is for a specific *work order type*.</span></span> <span data-ttu-id="ca141-113">Monet työtilaustyypit on liitetty lähdeasiakirjoihin, kuten osto- tai myyntitilauksiin.</span><span class="sxs-lookup"><span data-stu-id="ca141-113">Many work order types are associated with source documents, such as purchase or sales orders.</span></span> <span data-ttu-id="ca141-114">Muut työtilaustyypit kuitenkin edustavat erillisiä varastoprosesseja kuten inventointia.</span><span class="sxs-lookup"><span data-stu-id="ca141-114">However, other work order types represent separate warehouse processes, such as cycle counting.</span></span> <span data-ttu-id="ca141-115">*työpoolin tunnuksen* avulla voit järjestää työsi ryhmiin.</span><span class="sxs-lookup"><span data-stu-id="ca141-115">The *work pool ID* lets you organize work into groups.</span></span> 

<span data-ttu-id="ca141-116">Työotsikon määrityksen asetuksia voidaan käyttää määrittämään, milloin uusi työkappale tulisi luoda.</span><span class="sxs-lookup"><span data-stu-id="ca141-116">The settings in the work header definition can be used to determine when a new piece of work should be created.</span></span> <span data-ttu-id="ca141-117">Voit esimerkiksi määrittää keräilyrivien maksimimäärän ja odotetun keräilyajan maksimimäärän.</span><span class="sxs-lookup"><span data-stu-id="ca141-117">For example, you can set a maximum number of pick lines and a maximum expected pick time.</span></span> <span data-ttu-id="ca141-118">Tämän jälkeen, jos myyntitilauksen keräilyprosessin työ ylittää jommankumman näistä arvoista, työ jaetaan kahdeksi työkappaleeksi.</span><span class="sxs-lookup"><span data-stu-id="ca141-118">Then, if the work for a sales order picking process exceeds either of those values, that work is split into two pieces of work.</span></span> 

<span data-ttu-id="ca141-119">Työrivit edustavat fyysisiä tehtäviä, jotka vaaditaan työn käsittelemiseen.</span><span class="sxs-lookup"><span data-stu-id="ca141-119">The work lines represent the physical tasks that are required in order to process the work.</span></span> <span data-ttu-id="ca141-120">Esimerkiksi lähtevässä varastoprosessissa saattaa olla työrivi varastossa olevien nimikkeiden keräilyyn ja toinen rivi näiden nimikkeiden sijoittamiseen vaiheiden alueelle.</span><span class="sxs-lookup"><span data-stu-id="ca141-120">For example, for an outbound warehouse process, there might be a work line for picking up the items within the warehouse and another line for putting those items into a staging area.</span></span> <span data-ttu-id="ca141-121">Tämän jälkeen voi olla lisärivi nimikkeiden keräilyyn vaiheiden alueelta ja toinen rivi näiden nimikkeiden poispanoon kuorma-autoon osana lastausprosessia.</span><span class="sxs-lookup"><span data-stu-id="ca141-121">There can then be an additional line for picking the items from staging, and another line for putting the items into a truck as part of the loading process.</span></span> <span data-ttu-id="ca141-122">Voit määrittää *direktiivikoodin* työmallirivillä.</span><span class="sxs-lookup"><span data-stu-id="ca141-122">You can set a *directive code* on work template lines.</span></span> <span data-ttu-id="ca141-123">Direktiivikoodi linkitetään sijaintidirektiiviin ja se auttaa siten varmistamaan, että varastotyö käsitellään varaston oikeassa sijainnissa.</span><span class="sxs-lookup"><span data-stu-id="ca141-123">A directive code is linked to a location directive and therefore helps guarantee that the warehouse work is processed in the correct location in the warehouse.</span></span> 

<span data-ttu-id="ca141-124">Voit määrittää kyselyn ohjaamaan, milloin määrättyä työmallia käytetään.</span><span class="sxs-lookup"><span data-stu-id="ca141-124">You can set up a query to control when a particular work template is used.</span></span> <span data-ttu-id="ca141-125">Voit esimerkiksi määrittää rajoituksen siten, että määrättyä mallia voidaan käyttää työhön vain määrätyssä varastossa.</span><span class="sxs-lookup"><span data-stu-id="ca141-125">For example, you can set a limitation so that a particular template can be used for work only in a specific warehouse.</span></span> <span data-ttu-id="ca141-126">Vaihtoehtoisesti sinulla voi olla useita malleja, joita käytetään työn luomiseen lähtevän myyntitilauksen käsittelyyn riippuen myynnin alkuperästä.</span><span class="sxs-lookup"><span data-stu-id="ca141-126">Alternatively, you might have several templates that are used to create work for outbound sales order processing, depending on the sales origin.</span></span> <span data-ttu-id="ca141-127">Järjestelmä käyttää **järjestysnumero**-kenttää määrittämään käytettävissä olevien työmallien arviointijärjestyksen.</span><span class="sxs-lookup"><span data-stu-id="ca141-127">The system uses the **Sequence number** field to determine the order that the available work templates are assessed in.</span></span> <span data-ttu-id="ca141-128">Siksi, jos sinulla on erityinen kysely tietylle työmallille, anna sille alhainen järjestysnumero.</span><span class="sxs-lookup"><span data-stu-id="ca141-128">Therefore, if you have a very specific query for a particular work template, you should give it a low sequence number.</span></span> <span data-ttu-id="ca141-129">Tämä kysely arvioidaan sitten ennen muita, yleisempiä kyselyitä.</span><span class="sxs-lookup"><span data-stu-id="ca141-129">That query will then be evaluated before other, more general queries.</span></span> 

<span data-ttu-id="ca141-130">Voit lopettaa tai keskeyttää työprosessin työrivin **Pysäytä työ** -asetuksen avulla.</span><span class="sxs-lookup"><span data-stu-id="ca141-130">To stop or pause a work process, you can use the **Stop work** setting on the work line.</span></span> <span data-ttu-id="ca141-131">Siinä tapauksessa työn suorittavaa työntekijää ei pyydetä suorittamaan seuraavaa työrivin vaihetta.</span><span class="sxs-lookup"><span data-stu-id="ca141-131">In that case, the worker who is performing the work won't be asked to perform the next work line step.</span></span> <span data-ttu-id="ca141-132">Siirtyäkseen seuraavaan vaiheeseen, työntekijän on valittava työ uudelleen.</span><span class="sxs-lookup"><span data-stu-id="ca141-132">To move on to the next step, that worker or another worker must select the work again.</span></span> <span data-ttu-id="ca141-133">Voit myös erotella tehtävät työkappaleen sisällä käyttämällä eri *työluokan tunnusta* työmallin riveillä.</span><span class="sxs-lookup"><span data-stu-id="ca141-133">You can also separate the tasks within a piece of work by using a different *work class ID* on the work template lines.</span></span>

## <a name="location-directives"></a><span data-ttu-id="ca141-134">Sijaintidirektiivit</span><span class="sxs-lookup"><span data-stu-id="ca141-134">Location directives</span></span>

<span data-ttu-id="ca141-135">Sijaintidirektiivit ovat sääntöjä, jotka auttavat tunnistamaan keräily- ja poispanosijainnit varaston siirrossa.</span><span class="sxs-lookup"><span data-stu-id="ca141-135">Location directives are rules that help identify pick and put locations for inventory movement.</span></span> <span data-ttu-id="ca141-136">Esimerkiksi myyntitilaustapahtumassa sijaintidirektiivi määrittää, mistä nimikkeet kerätään ja minne kerätyt nimikkeet sijoitetaan.</span><span class="sxs-lookup"><span data-stu-id="ca141-136">For example, in a sales order transaction, a location directive determines where the items will be picked, and where the picked items will be put.</span></span> <span data-ttu-id="ca141-137">Sijaintidirektiivit koostuvat otsikosta ja siihen liittyvistä riveistä. Luot ne **Sijaintidirektiivi**-sivulla.</span><span class="sxs-lookup"><span data-stu-id="ca141-137">Location directives consist of a header and associated lines, and you create them on the **Location directives** page.</span></span> 

<span data-ttu-id="ca141-138">Otsikossa jokaisen direktiivin on oltava liitettynä *työtilaustyypille*. Se määrittää varastotapahtuman tyypin, johon direktiiviä tullaan käyttämään, kuten myyntitilaukset, täydennys, tai raaka-aineiden keräily.</span><span class="sxs-lookup"><span data-stu-id="ca141-138">On the header, each location directive must be associated with a *work order type* that specifies the type of inventory transaction that the directive will be used for, such as sales orders, replenishment, or raw material picking.</span></span> <span data-ttu-id="ca141-139">*työtyyppi* määrittää, käytetäänkö sijaintidirektiiviä keräily- tai poispanotyöhön, tai johonkin muuhun varastoprosessiin kuten laskentaan tai varaston tilan muutoksiin.</span><span class="sxs-lookup"><span data-stu-id="ca141-139">The *work type* specifies whether the location directive will be used for picking or putting work, or for some other warehouse process, such as counting or inventory status changes.</span></span> <span data-ttu-id="ca141-140">Sinun on myös määritettävä *toimipaikka* ja *varasto*.</span><span class="sxs-lookup"><span data-stu-id="ca141-140">You must also specify a *site* and a *warehouse*.</span></span> <span data-ttu-id="ca141-141">Otsikossa määrittämääsi *direktiivikoodia* voidaan käyttää linkittämään sijaintidirektiivin yhteen tai useampaan työmalliin.</span><span class="sxs-lookup"><span data-stu-id="ca141-141">A *directive code* that you specify on the header can be used to link the location directive to one or more work templates.</span></span>

<span data-ttu-id="ca141-145">Sijaintidirektiivin rivit asettavat lisärajoituksia sijainnin löytämissääntöjen käytölle.</span><span class="sxs-lookup"><span data-stu-id="ca141-145">The location directive lines set additional restrictions on the application of the location finding rules.</span></span> <span data-ttu-id="ca141-146">Voit määritellä vähimmäis- ja enimmäismäärät, joihin direktiiviä tulisi soveltaa ja voit määrittää, että direktiivi koskee määrättyä varastoyksikköä.</span><span class="sxs-lookup"><span data-stu-id="ca141-146">You can specify a minimum quantity and a maximum quantity that the directive should apply to, and you can specify that the directive should be for a specific inventory unit.</span></span> <span data-ttu-id="ca141-147">Esimerkiksi jos mittayksikkö on kuormalava, kuormalavojen nimikkeet voidaan asettaa tiettyyn paikkaan.</span><span class="sxs-lookup"><span data-stu-id="ca141-147">For example, if the unit of measure is pallets, the items in pallets can be put in a specific location.</span></span> <span data-ttu-id="ca141-148">Voit myös määrittää, voidaanko määrä jakaa eri sijainteihin.</span><span class="sxs-lookup"><span data-stu-id="ca141-148">You can also specify whether the quantity can be split across multiple locations.</span></span> <span data-ttu-id="ca141-149">Samoin kuin sijaintidirektiivin otsikolla, kullakin sijaintidirektiivin rivillä on järjestysnumero, joka määrittää näiden rivien arviointijärjestyksen.</span><span class="sxs-lookup"><span data-stu-id="ca141-149">Like the location directive header, each location directive line has a sequence number that determines the order that the lines are assessed in.</span></span>

<span data-ttu-id="ca141-150">Sijaintidirektiiveillä on yksi yksityiskohtien taso lisää: *sijaintidirektiivin toiminnot*.</span><span class="sxs-lookup"><span data-stu-id="ca141-150">Location directives have one additional level of detail: *location directive actions*.</span></span> <span data-ttu-id="ca141-151">Voit määrittää useita sijaintidirektiivin toimintoja kullekin riville.</span><span class="sxs-lookup"><span data-stu-id="ca141-151">You can define multiple location directive actions for each line.</span></span> <span data-ttu-id="ca141-152">Jälleen kerran järjestysnumeroa käytetään määrittämään järjestys, jossa arvioidaan toimia.</span><span class="sxs-lookup"><span data-stu-id="ca141-152">Once again, a sequence number is used to determine the order that the actions are assessed in.</span></span> <span data-ttu-id="ca141-153">Tällä tasolla voit määrittää kyselyn määrittääksesi, miten löydetään paras sijainti varastossa.</span><span class="sxs-lookup"><span data-stu-id="ca141-153">On this level, you can set up a query to define how to find the best location in the warehouse.</span></span> <span data-ttu-id="ca141-154">Voit myös käyttää esimääritettyjä **Strategia**-asetuksia parhaan sijainnin löytämiseen.</span><span class="sxs-lookup"><span data-stu-id="ca141-154">You can also use predefined **Strategy** settings to find an optimal location.</span></span>
