<span data-ttu-id="ea161-123">Työrivit ilmaisevat fyysisiä tehtäviä, jotka tarvitaan työn käsittelemiseen.</span><span class="sxs-lookup"><span data-stu-id="ea161-123">The work lines represent the physical tasks that are required to process the work.</span></span> <span data-ttu-id="ea161-124">Esimerkiksi lähtevässä varastoprosessissa saattaa olla yksi työrivi varastossa olevien nimikkeiden keräilyyn ja toinen rivi näiden nimikkeiden sijoittamiseen valmistelualueelle.</span><span class="sxs-lookup"><span data-stu-id="ea161-124">For example, for an outbound warehouse process, there might be one line for picking up the items in the warehouse and another line for putting those items into a staging area.</span></span> <span data-ttu-id="ea161-125">Tämän jälkeen voi olla lisärivi nimikkeiden keräilyyn valmistelualueelta ja toinen rivi näiden nimikkeiden asettamiseen kuorma-autoon lastausprosessin osana.</span><span class="sxs-lookup"><span data-stu-id="ea161-125">There can then be an additional line for picking the items from staging and another line for putting the items into a truck as part of the loading process.</span></span> <span data-ttu-id="ea161-126">Voit määrittää *direktiivikoodin* työmallirivillä.</span><span class="sxs-lookup"><span data-stu-id="ea161-126">You can set a *directive code* on work template lines.</span></span> <span data-ttu-id="ea161-127">Direktiivikoodi linkitetään sijaintidirektiiviin ja se auttaa siten varmistamaan, että varastotyö käsitellään varaston oikeassa sijainnissa.</span><span class="sxs-lookup"><span data-stu-id="ea161-127">A directive code is linked to a location directive, and therefore helps ensure that the warehouse work is processed in the correct location in the warehouse.</span></span>

<span data-ttu-id="ea161-128">Voit määrittää kyselyn ohjaamaan, milloin määrättyä työmallia käytetään.</span><span class="sxs-lookup"><span data-stu-id="ea161-128">You can set up a query to control when a particular work template is used.</span></span> <span data-ttu-id="ea161-129">Voit esimerkiksi määrittää rajoituksen siten, että määrättyä mallia voidaan käyttää työhön vain määrätyssä varastossa.</span><span class="sxs-lookup"><span data-stu-id="ea161-129">For example, you can set a limitation so that a particular template can be used for work only in a specific warehouse.</span></span> <span data-ttu-id="ea161-130">Vaihtoehtoisesti käytössä voi olla useita malleja, joita käytetään työn luomiseen lähtevän myyntitilauksen käsittelyyn riippuen myynnin alkuperästä.</span><span class="sxs-lookup"><span data-stu-id="ea161-130">Alternatively, you might have several templates that create work for outbound sales order processing, depending on the sales origin.</span></span> <span data-ttu-id="ea161-131">Järjestelmä käyttää **järjestysnumero**-kenttää määrittämään käytettävissä olevien työmallien arviointijärjestyksen.</span><span class="sxs-lookup"><span data-stu-id="ea161-131">The system uses the **Sequence number** field to determine the order that the available work templates are assessed in.</span></span> <span data-ttu-id="ea161-132">Siksi, jos sinulla on erityinen kysely tietylle työmallille, anna sille alhainen järjestysnumero.</span><span class="sxs-lookup"><span data-stu-id="ea161-132">Therefore, if you have a very specific query for a particular work template, you should give it a low sequence number.</span></span> <span data-ttu-id="ea161-133">Tämä kysely arvioidaan sitten ennen muita, yleisiä kyselyitä.</span><span class="sxs-lookup"><span data-stu-id="ea161-133">That query will then be evaluated before the other, more general queries.</span></span>

> [!NOTE]
> <span data-ttu-id="ea161-134">Voit estää järjestelmää korvaamasta automaattisesti työmallin *järjestysnumeroita* mallin poistamisen jälkeen ottamalla *Säilytä työmallin järjestysnumerot poistettaessa* -toiminto käyttöön [ominaisuuksien hallinnassa](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).</span><span class="sxs-lookup"><span data-stu-id="ea161-134">To prevent the system from automatically overwriting work template *sequence numbers* after a template has been deleted, turn on the *Preserve work template sequence numbers on delete* feature in [Feature management](../../fin-ops-core/fin-ops/get-started/feature-management/feature-management-overview.md).</span></span>

<span data-ttu-id="ea161-135">Voit lopettaa tai keskeyttää työprosessin työrivin **Pysäytä työ** -asetuksen avulla.</span><span class="sxs-lookup"><span data-stu-id="ea161-135">To stop or pause a work process, you can use the **Stop work** setting on the work line.</span></span> <span data-ttu-id="ea161-136">Siinä tapauksessa työn suorittavaa työntekijää ei pyydetä suorittamaan seuraavaa työrivin vaihetta.</span><span class="sxs-lookup"><span data-stu-id="ea161-136">In that case, the worker who is performing the work won't be asked to perform the next work line step.</span></span> <span data-ttu-id="ea161-137">Siirtyäkseen seuraavaan vaiheeseen, työntekijän on valittava työ uudelleen.</span><span class="sxs-lookup"><span data-stu-id="ea161-137">To move on to the next step, that worker or another worker must select the work again.</span></span> <span data-ttu-id="ea161-138">Voit myös erotella tehtävät työkappaleen sisällä käyttämällä eri *työluokan tunnusta* työmallin riveillä.</span><span class="sxs-lookup"><span data-stu-id="ea161-138">You can also separate the tasks within a piece of work by using a different *work class ID* on the work template lines.</span></span>

## <a name="location-directives"></a><span data-ttu-id="ea161-139">Sijaintidirektiivit</span><span class="sxs-lookup"><span data-stu-id="ea161-139">Location directives</span></span>

<span data-ttu-id="ea161-140">Sijaintidirektiivit ovat sääntöjä, jotka auttavat tunnistamaan keräily- ja poispanosijainnit varaston siirrossa.</span><span class="sxs-lookup"><span data-stu-id="ea161-140">Location directives are rules that help identify pick and put locations for inventory movement.</span></span> <span data-ttu-id="ea161-141">Esimerkiksi myyntitilaustapahtumassa sijaintidirektiivi määrittää, mistä nimikkeet kerätään ja minne kerätyt nimikkeet sijoitetaan.</span><span class="sxs-lookup"><span data-stu-id="ea161-141">For example, in a sales order transaction, a location directive determines where the items will be picked, and where the picked items will be put.</span></span> <span data-ttu-id="ea161-142">Sijaintidirektiivit koostuvat otsikosta ja siihen liittyvistä riveistä. Luot ne **Sijaintidirektiivi**-sivulla.</span><span class="sxs-lookup"><span data-stu-id="ea161-142">Location directives consist of a header and associated lines, and you create them on the **Location directives** page.</span></span>

<span data-ttu-id="ea161-143">Otsikossa jokaisen direktiivin on oltava liitettynä *työtilaustyypille*. Se määrittää varastotapahtuman tyypin, johon direktiiviä tullaan käyttämään, kuten myyntitilaukset, täydennys, tai raaka-aineiden keräily.</span><span class="sxs-lookup"><span data-stu-id="ea161-143">On the header, each location directive must be associated with a *work order type* that specifies the type of inventory transaction that the directive will be used for, such as sales orders, replenishment, or raw material picking.</span></span> <span data-ttu-id="ea161-144">*työtyyppi* määrittää, käytetäänkö sijaintidirektiiviä keräily- tai poispanotyöhön, tai johonkin muuhun varastoprosessiin kuten laskentaan tai varaston tilan muutoksiin.</span><span class="sxs-lookup"><span data-stu-id="ea161-144">The *work type* specifies whether the location directive will be used for picking or putting work, or for some other warehouse process, such as counting or inventory status changes.</span></span> <span data-ttu-id="ea161-145">Sinun on myös määritettävä *toimipaikka* ja *varasto*.</span><span class="sxs-lookup"><span data-stu-id="ea161-145">You must also specify a *site* and a *warehouse*.</span></span> <span data-ttu-id="ea161-146">Otsikossa määrittämääsi *direktiivikoodia* voidaan käyttää linkittämään sijaintidirektiivin yhteen tai useampaan työmalliin.</span><span class="sxs-lookup"><span data-stu-id="ea161-146">A *directive code* that you specify on the header can be used to link the location directive to one or more work templates.</span></span>
