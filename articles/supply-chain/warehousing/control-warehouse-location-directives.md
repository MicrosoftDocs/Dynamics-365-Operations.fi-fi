## <a name="work-templates"></a><span data-ttu-id="4c5ec-108">Työmallit</span><span class="sxs-lookup"><span data-stu-id="4c5ec-108">Work templates</span></span>

<span data-ttu-id="4c5ec-124">Voit määrittää kyselyn ohjaamaan, milloin määrättyä työmallia käytetään.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-124">You can set up a query to control when a particular work template is used.</span></span> <span data-ttu-id="4c5ec-125">Voit esimerkiksi määrittää rajoituksen siten, että määrättyä mallia voidaan käyttää työhön vain määrätyssä varastossa.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-125">For example, you can set a limitation so that a particular template can be used for work only in a specific warehouse.</span></span> <span data-ttu-id="4c5ec-126">Vaihtoehtoisesti sinulla voi olla useita malleja, joita käytetään työn luomiseen lähtevän myyntitilauksen käsittelyyn riippuen myynnin alkuperästä.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-126">Alternatively, you might have several templates that are used to create work for outbound sales order processing, depending on the sales origin.</span></span> <span data-ttu-id="4c5ec-127">Järjestelmä käyttää **järjestysnumero**-kenttää määrittämään käytettävissä olevien työmallien arviointijärjestyksen.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-127">The system uses the **Sequence number** field to determine the order that the available work templates are assessed in.</span></span> <span data-ttu-id="4c5ec-128">Siksi, jos sinulla on erityinen kysely tietylle työmallille, anna sille alhainen järjestysnumero.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-128">Therefore, if you have a very specific query for a particular work template, you should give it a low sequence number.</span></span> <span data-ttu-id="4c5ec-129">Tämä kysely arvioidaan sitten ennen muita, yleisempiä kyselyitä.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-129">That query will then be evaluated before other, more general queries.</span></span> 

<span data-ttu-id="4c5ec-130">Voit lopettaa tai keskeyttää työprosessin työrivin **Pysäytä työ** -asetuksen avulla.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-130">To stop or pause a work process, you can use the **Stop work** setting on the work line.</span></span> <span data-ttu-id="4c5ec-131">Siinä tapauksessa työn suorittavaa työntekijää ei pyydetä suorittamaan seuraavaa työrivin vaihetta.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-131">In that case, the worker who is performing the work won't be asked to perform the next work line step.</span></span> <span data-ttu-id="4c5ec-132">Siirtyäkseen seuraavaan vaiheeseen, työntekijän on valittava työ uudelleen.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-132">To move on to the next step, that worker or another worker must select the work again.</span></span> <span data-ttu-id="4c5ec-133">Voit myös erotella tehtävät työkappaleen sisällä käyttämällä eri *työluokan tunnusta* työmallin riveillä.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-133">You can also separate the tasks within a piece of work by using a different *work class ID* on the work template lines.</span></span>

## <a name="location-directives"></a><span data-ttu-id="4c5ec-134">Sijaintidirektiivit</span><span class="sxs-lookup"><span data-stu-id="4c5ec-134">Location directives</span></span>

Luot ne **Sijaintidirektiivi**-sivulla.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-137">Location directives consist of a header and associated lines, and you create them on the **Location directives** page.</span></span> 

<span data-ttu-id="4c5ec-138">Otsikossa jokaisen direktiivin on oltava liitettynä *työtilaustyypille*. <span data-ttu-id="4c5ec-156">Tätä esimerkkiä varten tarkastelemme ostotilausprosessia, jossa sijaintidirektiivin on löydettävä varastosta vapaata kapasiteettia varastonimikkeille, jotka on juuri rekisteröity vastaanottolaiturilla.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-156">For this example, we will consider a purchase order process where the location directive must find free capacity within a warehouse for inventory items that have just been registered at the receiving dock.</span></span> <span data-ttu-id="4c5ec-157">Ensin haluamme yrittää löytää varastosta vapaata kapasiteettia konsolidoimalla nykyiseen käsillä olevaan varastoon.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-157">First, we want to try to find free capacity within the warehouse by consolidating with existing on-hand inventory.</span></span> <span data-ttu-id="4c5ec-158">Jos konsolidointi ei ole mahdollista, haluamme löytää tyhjän sijainnin.</span><span class="sxs-lookup"><span data-stu-id="4c5ec-158">If consolidation isn't possible, we then want to find an empty location.</span></span>
