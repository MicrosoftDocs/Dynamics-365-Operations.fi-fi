# <a name="replenishment-overview"></a><span data-ttu-id="d91a2-103">Täydennys – yleiskatsaus</span><span class="sxs-lookup"><span data-stu-id="d91a2-103">Replenishment overview</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="d91a2-104">Tässä ohjeaiheessa käsitellään täydennysstrategioita, joita voi käyttää varastonhallinnan toimintoja käyttävissä varastoissa.</span><span class="sxs-lookup"><span data-stu-id="d91a2-104">This topic describes the replenishment strategies that are available for warehouses that use the functionality that is available in Warehouse management.</span></span> <span data-ttu-id="d91a2-105">Tiedot eivät koske Inventoinnin- ja varastonhallinnassa käytettävissä olevaa varastoratkaisua.</span><span class="sxs-lookup"><span data-stu-id="d91a2-105">The information in this topic doesn't apply to the warehousing solution that is available in Inventory management.</span></span>

<span data-ttu-id="d91a2-106">Seuraavat täydennysstrategiat ovat käytettävissä:</span><span class="sxs-lookup"><span data-stu-id="d91a2-106">The following replenishment strategies are available:</span></span>

- <span data-ttu-id="d91a2-107">**Aallon kysynnän täydennys** – Tämä strategia luo lähteville tilausten tai kuormien täydennystyön, jos varasto ei ole käytettävissä, kun aalto luo työt.</span><span class="sxs-lookup"><span data-stu-id="d91a2-107">**Wave demand replenishment** – This strategy creates replenishment work for outbound orders or loads if inventory isn't available when the wave creates work.</span></span> <span data-ttu-id="d91a2-108">Täydennystyö voidaan esimerkiksi luoda, jos myyntitilaukseen vaadittava määrä ei ole käytettävissä, kun aalto käsitellään.</span><span class="sxs-lookup"><span data-stu-id="d91a2-108">For example, replenishment work can be created if the quantity that is required for a sales order isn't available when a wave is processed.</span></span>
- <span data-ttu-id="d91a2-109">**Vähimmäis- tai enimmäistäydennys** – Tässä strategiassa varastoinnin vähimmäis-ja enimmäisrajoilla määritetään, milloin sijainnit on täydennettävä.</span><span class="sxs-lookup"><span data-stu-id="d91a2-109">**Min/Max replenishment** – This strategy uses minimum and maximum stocking limits to determine when locations should be replenished.</span></span> <span data-ttu-id="d91a2-110">Nimike ja sijaintiehto määrittävät varaston, joka arvioidaan täydennystä varten.</span><span class="sxs-lookup"><span data-stu-id="d91a2-110">The item and location criteria define the inventory that is evaluated for replenishment.</span></span> <span data-ttu-id="d91a2-111">Vähimmäis- tai enimmäistäydennykset malli ovat ensisijainen mekanismi optimaalisten tasojen ylläpitämiseksi keräilysijainneissa.</span><span class="sxs-lookup"><span data-stu-id="d91a2-111">Min/Max replenishment templates are the primary mechanism for maintaining optimal levels in picking locations.</span></span> <span data-ttu-id="d91a2-112">Voit kysynnän täydennyksen avulla varmistaa, että varastoa riittää aallon kysyntään vastaamiseen. Tämä toimii vähimmäis- ja enimmäistäydennysjaksojen välisenä täydennyksenä:</span><span class="sxs-lookup"><span data-stu-id="d91a2-112">To help guarantee that enough pick face inventory is available to meet wave demand, you can use demand replenishment as a supplement between Min/Max replenishment cycles.</span></span>
- <span data-ttu-id="d91a2-113">**Kuorman kysynnän täydennys** – Tässä strategiassa summataan useiden kuormien kysyntä ja luodaan täydennystyö, jota tarvitaan varaston täyttämiseen liittyvissä keräilysijainneissa.</span><span class="sxs-lookup"><span data-stu-id="d91a2-113">**Load demand replenishment** – This strategy sums the demand for several loads and creates the replenishment work that is required in order to stock the relevant picking locations.</span></span> <span data-ttu-id="d91a2-114">Tämä strategia auttaa varmistamaan, että luodut kuormat voidaan kerätä varastosta vapauttamisen jälkeen.</span><span class="sxs-lookup"><span data-stu-id="d91a2-114">This strategy helps guarantee that the loads that are created can be picked in the warehouse after they are released.</span></span>
- <span data-ttu-id="d91a2-115">**Välitön täydennys** – tällä menetelmällä varasto täydennetään ennen kuin aalto suoritetaan, jos kohdistus epäonnistuu sijaintidirektiivin riville, jolla on täydennysmalli.</span><span class="sxs-lookup"><span data-stu-id="d91a2-115">**Immediate replenishment** – This strategy replenishes inventory before a wave is run if allocation fails for a location directive line that has a replenishment template.</span></span> 

<span data-ttu-id="d91a2-116">Täydennystyö luodaan täydennysmallin perusteella kaikissa neljässä strategiassa.</span><span class="sxs-lookup"><span data-stu-id="d91a2-116">All four strategies create replenishment work, based on a replenishment template.</span></span>

## <a name="wave-demand-replenishment"></a><span data-ttu-id="d91a2-117">Aallon kysynnän täydennys</span><span class="sxs-lookup"><span data-stu-id="d91a2-117">Wave demand replenishment</span></span>
<span data-ttu-id="d91a2-118">Aallon kysynnän täydennys luo kysynnän mukaan täydennystyön, jos tuotantotilausten, kanbanien, lähtevien tilausten tai kuormien määrä ei ole käytettävissä, kun aalto luo työn.</span><span class="sxs-lookup"><span data-stu-id="d91a2-118">Wave demand replenishment creates replenishment work, based on demand, if the quantity that is required for production orders, kanbans, outbound orders, or loads isn't available when a wave creates work.</span></span> <span data-ttu-id="d91a2-119">Täydennysmallissa on tietoja nimikkeen ehdoista, mittayksiköstä, kysynnän lisäyksestä ja sijainnista.</span><span class="sxs-lookup"><span data-stu-id="d91a2-119">The replenishment template contains information about the item criteria, the unit of measure, the demand increment, and the location.</span></span> 

<span data-ttu-id="d91a2-120">Sijaintidirektiiveillä määritetään, mitä sijaintia on täydennettävä.</span><span class="sxs-lookup"><span data-stu-id="d91a2-120">Location directives are used to determine which location should be replenished.</span></span> <span data-ttu-id="d91a2-121">Nämä sijaintidirektiivit linkitetään täydennysmalliin **Direktiivikoodi**-kentässä.</span><span class="sxs-lookup"><span data-stu-id="d91a2-121">You link these location directives to the replenishment template by using the **Directive code** field.</span></span> <span data-ttu-id="d91a2-122">Jos **Direktiivikoodi**-kenttää ei ole määritetty, käytettävä suuntadirektiivi määritetään kyselyillä.</span><span class="sxs-lookup"><span data-stu-id="d91a2-122">If the **Directive code** field isn't set, queries are used to determine which location directive should be used.</span></span> <span data-ttu-id="d91a2-123">Huomaa, että jos direktiivikoodia ei ole määritetty täydennysmallissa ja sijaintidirektiivillä on direktiivikoodi, sijaintidirektiivi ohitetaan, vaikka sijaintidirektiivikysely olisi oikein.</span><span class="sxs-lookup"><span data-stu-id="d91a2-123">Note that if a directive code isn't specified in the replenishment template, and the location directive has a directive code, the location directive will be ignored, even if the query on the location directive is correct.</span></span> <span data-ttu-id="d91a2-124">Keräilysijaintidirektiivillä määritetään, mistä varaston täydennys hankitaan.</span><span class="sxs-lookup"><span data-stu-id="d91a2-124">Pick location directives are used to determine where to get inventory for the replenishment.</span></span>

<span data-ttu-id="d91a2-133">**Täydennysyksikkö** on täydennyksen vähimmäisyksikkö.</span><span class="sxs-lookup"><span data-stu-id="d91a2-133">**Replenishment unit** is the minimum unit to replenish.</span></span> <span data-ttu-id="d91a2-134">Tämän on oltava kokonaisluku, joka on yksikön kerrannainen.</span><span class="sxs-lookup"><span data-stu-id="d91a2-134">This must be a whole number that is a multiple of the unit.</span></span> <span data-ttu-id="d91a2-135">Järjestelmä pyöristää ylöspäin suurimpaan mahdolliseen yksikköön työtä luotaessa.</span><span class="sxs-lookup"><span data-stu-id="d91a2-135">The system will round up to the highest unit possible when creating work.</span></span>

<span data-ttu-id="d91a2-136">Kysynnän täydennystä tuetaan myyntitilauksissa, siirtotilauksissa, tuotantotilauksissa ja kanbaneissa.</span><span class="sxs-lookup"><span data-stu-id="d91a2-136">Demand replenishment is supported for sales orders, transfer orders, production orders, and kanbans.</span></span> 

## <a name="minmax-replenishment"></a><span data-ttu-id="d91a2-137">Vähimmäis- tai enimmäistäydennys</span><span class="sxs-lookup"><span data-stu-id="d91a2-137">Min/Max replenishment</span></span>
<span data-ttu-id="d91a2-138">Vähimmäis- tai enimmäistäydennyksessä varastoa täydennetään niin, että on määritettyjen vähimmäis- ja enimmäisrajojen välillä.</span><span class="sxs-lookup"><span data-stu-id="d91a2-138">In Min/Max replenishment, stock is replenished so that it's between the minimum and maximum limits that have been set.</span></span> <span data-ttu-id="d91a2-139">Yleensä tämä prosessi tapahtuu kerran päivässä, mikä auttaa varmistamaan, että kaikki keräilysijainnit on täytetty enimmäistasolle ennen keräilyn aloittamista.</span><span class="sxs-lookup"><span data-stu-id="d91a2-139">Typically, this process occurs one time every day, to help guarantee that all picking locations are filled to the maximum level before picking starts.</span></span> 

<span data-ttu-id="d91a2-140">Vähimmäis- ja enimmäissummat määritetään täydennysmallissa.</span><span class="sxs-lookup"><span data-stu-id="d91a2-140">The minimum and maximum amounts are set in a replenishment template.</span></span> <span data-ttu-id="d91a2-141">Monet muut mallin asetukset muistuttavat aallon kysynnän täydennyksessä käytettäviä malleja.</span><span class="sxs-lookup"><span data-stu-id="d91a2-141">Many of the other settings in the template resemble the settings in templates that are used in Wave demand replenishment.</span></span> <span data-ttu-id="d91a2-142">Mallissa on oltava yksi rivi kullekin nimikkeelle ja sijainnille.</span><span class="sxs-lookup"><span data-stu-id="d91a2-142">The template should contain one line for each item and location.</span></span> <span data-ttu-id="d91a2-143">Kun suoritat täydennyksen käyttämällä erätyötä, järjestelmä tarkistaa täydennyksen tarpeen siinä järjestyksessä, johon rivit on järjestetty.</span><span class="sxs-lookup"><span data-stu-id="d91a2-143">When you run replenishment by using the batch job, the system evaluates whether replenishment is required by using the sequence that the lines are organized in.</span></span>
