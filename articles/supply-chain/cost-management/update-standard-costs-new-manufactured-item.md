# <a name="update-standard-costs-for-a-new-manufactured-item"></a><span data-ttu-id="f0185-103">Uuden valmistetun nimikkeen standardikustannusten päivittäminen</span><span class="sxs-lookup"><span data-stu-id="f0185-103">Update standard costs for a new manufactured item</span></span>

[!INCLUDE [banner](../includes/banner.md)]

<span data-ttu-id="f0185-104">Tämä artikkeli sisältää ohjeita uuden valmistetun tuotteen standardikustannusten päivittämiseen.</span><span class="sxs-lookup"><span data-stu-id="f0185-104">This article provides guidance for updating standard costs for a new manufactured item.</span></span> 

<span data-ttu-id="f0185-105">Näissä ohjeissa lähdetään siitä, että standardikustannusten päivityksissä käytetään kahden version mallia.</span><span class="sxs-lookup"><span data-stu-id="f0185-105">The following guidelines assume that you use a two-version approach to update standard costs.</span></span> <span data-ttu-id="f0185-106">Kahden version mallissa yksi kustannuslaskelmaversio sisältää alun perin määritetyt lukitun kauden standardikustannukset ja toinen kustannuslaskelmaversio uusia valmistettuja nimikkeitä koskevat lisäpäivitykset.</span><span class="sxs-lookup"><span data-stu-id="f0185-106">In this approach, one costing version contains the standard costs that were originally defined for the frozen period, and the second costing version contains the incremental updates that pertain to the new manufactured items.</span></span> <span data-ttu-id="f0185-107">Lisäpäivitykset lisätään kustannustietueina toiseen kustannuslaskelmaversioon ja lopuksi ne myös aktivoidaan.</span><span class="sxs-lookup"><span data-stu-id="f0185-107">The incremental updates are entered as cost records in the second costing version, and eventually they are enabled.</span></span> <span data-ttu-id="f0185-108">Kahden version lähestymistapa edellyttää, että määritetään toinen kustannuslaskelmaversio.</span><span class="sxs-lookup"><span data-stu-id="f0185-108">The two-version approach requires that you define a second costing version.</span></span> <span data-ttu-id="f0185-109">Tämän kustannuslaskelmaversion määritysohjeet:</span><span class="sxs-lookup"><span data-stu-id="f0185-109">Here are the guidelines for defining this costing version:</span></span>

-   <span data-ttu-id="f0185-110">Määritä **standardikustannusten** kustannuslaskelmatyyppi.</span><span class="sxs-lookup"><span data-stu-id="f0185-110">Assign a costing type of **Standard cost**.</span></span>
-   <span data-ttu-id="f0185-111">Määritä kustannuslaskelmaversiolle sisältöä kuvaava tunnus, kuten **2016-PÄIVITYKSET**.</span><span class="sxs-lookup"><span data-stu-id="f0185-111">Assign a significant identifier that indicates the contents of the costing version, such as **2016-UPDATES**.</span></span>
-   <span data-ttu-id="f0185-112">Varmista **Salli hintatyyppi** -kenttäryhmässä, että **Kustannushinta**-kohdan arvoksi on asetettu **Kyllä**.</span><span class="sxs-lookup"><span data-stu-id="f0185-112">In the **Allow price types** field group, make sure that **Cost price** is set to **Yes**.</span></span>
-   <span data-ttu-id="f0185-113">Salli kustannustietueiden lisääminen kaikille toimipaikoille (eli jätä **Toimipaikka**-kenttä tyhjäksi).</span><span class="sxs-lookup"><span data-stu-id="f0185-113">Allow cost records to be entered for all sites (that is, leave the **Site** field blank).</span></span> <span data-ttu-id="f0185-114">Jos lisäät toimipaikan, kustannustietueita voi lisätä vain kyseiselle toimipaikalle.</span><span class="sxs-lookup"><span data-stu-id="f0185-114">If you enter a site, cost records can be entered only for that site.</span></span>
-   <span data-ttu-id="f0185-115">Käytä varmistusperiaatetta **Aktiivinen**.</span><span class="sxs-lookup"><span data-stu-id="f0185-115">Use a fallback principle of **Active**.</span></span>

<span data-ttu-id="f0185-116">Jos haluat lisätä uusia valmistettavia nimikkeitä lukitun kauden eri kohtiin, noudata seuraavia ohjeita.</span><span class="sxs-lookup"><span data-stu-id="f0185-116">To add new manufacturing items throughout the frozen period, follow these steps.</span></span>

1.  <span data-ttu-id="f0185-117">Ota käyttöön kustannustietojen lisääminen toiseen, lisäpäivitykset sisältävään kustannuslaskelmaan **Kustannuslaskentaversion määritys** -sivulla.</span><span class="sxs-lookup"><span data-stu-id="f0185-117">Use the **Costing version setup** page to enable cost records to be entered into the second costing version that contains the incremental updates.</span></span> <span data-ttu-id="f0185-118">Estä odottavien kustannusten aktivointi siten, että aktivointi sallitaan vasta, kun odottavat kustannukset on määritetty kokonaisuudessaan ja totuudenmukaisesti.</span><span class="sxs-lookup"><span data-stu-id="f0185-118">Prevent the activation of pending costs, where activation is allowed after pending costs have been completely and accurately defined.</span></span> <span data-ttu-id="f0185-119">Määritä tyhjä alkamispäivä menettelyksi kustannuslaskelmaversioon ja lisää sitten aloituspäivämäärä kunkin kustannuslaskelman lisäämisen yhteydessä.</span><span class="sxs-lookup"><span data-stu-id="f0185-119">Indicate a blank from-date as a policy in the costing version, and then enter the from-date when you enter each cost record.</span></span> <span data-ttu-id="f0185-120">Aloituspäivämäärän pitäisi vastata päivämäärää, joka sijoittuu uusien nimikkeiden ostamista tai valmistamista edeltävälle ajalle.</span><span class="sxs-lookup"><span data-stu-id="f0185-120">The from-date should represent a date before the new items are purchased or manufactured.</span></span>

3.  <span data-ttu-id="f0185-123">Laske uusien valmistettujen nimikkeiden kustannukset **Laskenta**-sivulla.</span><span class="sxs-lookup"><span data-stu-id="f0185-123">Calculate the cost of new manufactured items by using the **Calculation** page.</span></span> <span data-ttu-id="f0185-124">Avaa **Laskenta** -sivu **Kustannuslaskentaversion ylläpito** -sivulta ja valitse kustannuslaskentaversio, joka sisältää lisäpäivitykset.</span><span class="sxs-lookup"><span data-stu-id="f0185-124">Open the **Calculation** page from the **Costing version maintenance** page, and then select the costing version that contains the incremental updates.</span></span> <span data-ttu-id="f0185-125">Valintaehtojen avulla voit määrittää uuden valmistetun nimikkeen ja jonkin sen valmistetuista komponenteista.</span><span class="sxs-lookup"><span data-stu-id="f0185-125">Use the selection criteria to specify the new manufactured item and any one of its manufactured components.</span></span> <span data-ttu-id="f0185-126">Monitasoisessa tuoterakenteessa saatat joutua määrittämään päänimikkeen, jonka komponenttina on uusia valmistettuja nimikkeitä.</span><span class="sxs-lookup"><span data-stu-id="f0185-126">In a multilevel product structure, you might also have to specify any parent item that contains the new manufactured items as a component.</span></span> <span data-ttu-id="f0185-127">Lisää tuoterakennelaskelmalle aloituspäivämäärä, joka vastaa uusien valmistettujen nimikkeiden valmistuksen aloittamista.</span><span class="sxs-lookup"><span data-stu-id="f0185-127">Enter a from-date for the bill of materials (BOM) calculation that corresponds to the start of manufacturing for the new manufactured items.</span></span> <span data-ttu-id="f0185-128">Aloituspäivämäärän pitää olla nimikkeen tuoterakenneversion ja reititysversion voimassaolopäivien puitteissa.</span><span class="sxs-lookup"><span data-stu-id="f0185-128">The from-date should be in the effective dates for the item’s BOM version and route version.</span></span> <span data-ttu-id="f0185-129">**Huomautus:** Puuttuva kustannustietue voi tarkoittaa uutta valmistettua nimikettä.</span><span class="sxs-lookup"><span data-stu-id="f0185-129">**Note:** A missing cost record can indicate a new manufactured item.</span></span> <span data-ttu-id="f0185-130">Tuoterakennelaskelmat voivat rajoittua nimikkeisiin, joilla on puuttuvia kustannuksia.</span><span class="sxs-lookup"><span data-stu-id="f0185-130">BOM calculations can be limited to items that have missing costs.</span></span>
4.  <span data-ttu-id="f0185-131">Tarkista, että uusien valmistettujen nimikkeiden lasketut kustannukset ovat täydelliset ja totuudenmukaiset.</span><span class="sxs-lookup"><span data-stu-id="f0185-131">Verify the completeness and accuracy of the calculated costs for new manufactured items.</span></span> <span data-ttu-id="f0185-132">**Valmis**-sivun avulla voit tarkistaa kunkin kustannustietueen lasketut kustannukset sekä mahdolliset Infoloki-varoitussanomat.</span><span class="sxs-lookup"><span data-stu-id="f0185-132">Use the **Complete** page to review the calculated costs for each item cost record, and also review any Infolog warning messages.</span></span> <span data-ttu-id="f0185-133">Voit käyttää laskettujen kustannusten tarkistamiseen myös **Laskenta**-sivua.</span><span class="sxs-lookup"><span data-stu-id="f0185-133">Alternatively, use the **Calculation** page to review the calculated costs.</span></span>
5.  <span data-ttu-id="f0185-134">Muuta estomerkkiä **Kustannuslaskentaversion määritys** -sivulla siten, että toiseen kustannuslaskelmaversioon lisätyt odottavat kustannustietueet voidaan aktivoida.</span><span class="sxs-lookup"><span data-stu-id="f0185-134">Use the **Costing version setup** page to change the blocking flag to allow activation of the pending cost records in the second costing version.</span></span>

7.  <span data-ttu-id="f0185-137">Voit estää muiden tietojen ylläpidon muuttamalla toiseen kustannuslaskentaversioon lisättyjä estomerkintöjä **Kustannuslaskentaversion määritys** -sivulla.</span><span class="sxs-lookup"><span data-stu-id="f0185-137">Use the **Costing version setup** page to change the blocking flags in the second costing version to prevent additional data maintenance.</span></span> <span data-ttu-id="f0185-138">Estomenettelyt estävät uusien odottavien kustannusten syötön ja odottavien kustannusten aktivoinnin.</span><span class="sxs-lookup"><span data-stu-id="f0185-138">The blocking policies prevent the entry of new pending costs and the activation of pending costs.</span></span>
