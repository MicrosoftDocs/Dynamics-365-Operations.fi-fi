# <a name="order-promising"></a><span data-ttu-id="f3f58-104">Luvatut tilaukset</span><span class="sxs-lookup"><span data-stu-id="f3f58-104">Order promising</span></span>

<span data-ttu-id="f3f58-107">Luvatuissa tilauksissa lasketaan, mikä on aikaisin lähetys- ja vastaanottopäivä toimituspäivän valvontapäivän ja kuljetuspäivien perusteella.</span><span class="sxs-lookup"><span data-stu-id="f3f58-107">Order promising calculates the earliest ship and receipt dates, and is based on the delivery date control method and transport days.</span></span> <span data-ttu-id="f3f58-108">Valittavana on neljä toimituspäivän valvontamenetelmää:</span><span class="sxs-lookup"><span data-stu-id="f3f58-108">You can select among four delivery date control methods:</span></span>

## <a name="atp-calculations"></a><span data-ttu-id="f3f58-121">ATP-laskelmat</span><span class="sxs-lookup"><span data-stu-id="f3f58-121">ATP calculations</span></span>
<span data-ttu-id="f3f58-122">ATP-määrä lasketaan "ennakoivan kumulatiivisen ATP-määrän" avulla.</span><span class="sxs-lookup"><span data-stu-id="f3f58-122">The ATP quantity is calculated by using the “cumulative ATP with look-ahead” method.</span></span> <span data-ttu-id="f3f58-123">Tämän ATP-laskentamenetelmän pääasiallinen etu on mahdollisuus käsitellä myös tilanteet, joissa vastaanottojen välisten varasto-ottojen summa on suurempi kuin uusin vastaanotto (esim. kun tarpeen täyttäminen edellyttää aiemman vastaanoton määrän käyttämistä).</span><span class="sxs-lookup"><span data-stu-id="f3f58-123">The main advantage of this ATP calculation method is that it can handle cases where the sum of issues among receipts is more than the latest receipt (for example, when a quantity from an earlier receipt must be used to meet a requirement).</span></span> <span data-ttu-id="f3f58-124">Ennakoiva kumulatiivinen ATP -laskentatapa sisällyttää kaikki varasto-otot, kunnes vastaanotettavien kumulatiivinen määrä ylittää otettavien kumulatiivisen määrän.</span><span class="sxs-lookup"><span data-stu-id="f3f58-124">The “cumulative ATP with look-ahead” calculation method includes all issues until the cumulative quantity to receive exceeds the cumulative quantity to issue.</span></span> <span data-ttu-id="f3f58-125">Niinpä tämä ATP-laskentatapa arvioi, voidaanko osaa aiemman kauden määrästä käyttää myöhemmällä kaudella.</span><span class="sxs-lookup"><span data-stu-id="f3f58-125">Therefore, this ATP calculation method evaluates whether some of the quantity from an earlier period can be used in a later period.</span></span>  

<span data-ttu-id="f3f58-126">ATP-määrä on ensimmäisen kauden varaamaton varastosaldo.</span><span class="sxs-lookup"><span data-stu-id="f3f58-126">The ATP quantity is the uncommitted inventory balance in the first period.</span></span> <span data-ttu-id="f3f58-127">Tavallisesti se lasketaan jokaiselle kaudelle, jolle on ajoitettu vastaanotto.</span><span class="sxs-lookup"><span data-stu-id="f3f58-127">Typically, it's calculated for each period in which a receipt is scheduled.</span></span> <span data-ttu-id="f3f58-128">Ohjelma laskee ATP-kauden päivinä ja käyttää kuluvaa päivää ATP-määrän ensimmäisen päivämääränä.</span><span class="sxs-lookup"><span data-stu-id="f3f58-128">The program calculates the ATP period in days and calculates the current date as the first date for the ATP quantity.</span></span> <span data-ttu-id="f3f58-129">Ensimmäisellä kaudella ATP ottaa mukaan käytettävissä olevan varaston, josta vähennetään erääntyvät ja erääntyneet asiakastilaukset.</span><span class="sxs-lookup"><span data-stu-id="f3f58-129">In the first period, ATP includes on-hand inventory minus customer orders that are due and overdue.</span></span>  

<span data-ttu-id="f3f58-130">ATP-määrä lasketaan seuraavaa kaavaa käyttäen:</span><span class="sxs-lookup"><span data-stu-id="f3f58-130">ATP is calculated by using the following formula:</span></span>  

<span data-ttu-id="f3f58-131">ATP = edeltävän kauden ATP + nykyisen kauden vastaanotot – nykyisen kauden varasto-otot – jokaisen tulevan kauden varasto-ottojen nettosumma siihen kauteen saakka, jonka kohdalla kaikkien tulevien kausien vastaanottojen summa kyseinen tuleva kausi mukaan lukien on suurempi kuin varasto-ottojen summa kyseinen tuleva kausi mukaan lukien.</span><span class="sxs-lookup"><span data-stu-id="f3f58-131">ATP = ATP for the previous period + Receipts for the current period – Issues for the current period – Net issue quantity for each future period until the period when the sum of receipts for all future periods, up to and including the future period, exceeds the sum of issues up to and including the future period.</span></span>  

<span data-ttu-id="f3f58-132">Huomaa, että ATP-laskennassa ei ole tietoja erääntymispäivästä eikä ATP-aikarajan jälkeisistä tiedoista, joita järjestelmä odottaa, kun mitään määrää voidaan luvata.</span><span class="sxs-lookup"><span data-stu-id="f3f58-132">Notice that the ATP calculation does not include information around expiry date and beyond the ATP time fence that the system expects when any quantity can be promised.</span></span>

<span data-ttu-id="f3f58-133">Kun huomioitavia varasto-ottoja tai vastaanottoja ei enää ole, seuraavien päivämäärien ATP-määrä on sama kuin viimeinen laskettu ATP-määrä.</span><span class="sxs-lookup"><span data-stu-id="f3f58-133">When there are no more issues or receipts to consider, the ATP quantity for the following dates is the same as the latest calculated ATP quantity.</span></span>  

<span data-ttu-id="f3f58-134">Jos kaikkia nimikkeessä käytettäviä dimensioita ei anneta, kun ATP-tarkistus on valmis, ne voidaan silti määrittää varasto-ottoon ja vastaanottoihin.</span><span class="sxs-lookup"><span data-stu-id="f3f58-134">If not all the dimensions that are used for an item are given when the ATP check is completed, they can still be specified on the issue and receipts.</span></span> <span data-ttu-id="f3f58-135">Tässä tilanteessa ATP-laskenta edellyttää, että vastaanotot ja varasto-otot on koottava valmiisiin dimensioihin, jotta ATP-laskennassa käytettävien vastaanotto- ja varasto-ottorivien määrää voidaan vähentää.</span><span class="sxs-lookup"><span data-stu-id="f3f58-135">In this case, in the ATP calculation, the receipts and issues must be aggregated to the existing dimensions to reduce the number of receipt and issue lines that are used in the ATP calculation.</span></span>  

<span data-ttu-id="f3f58-136">Näytettävä ATP-määrä on aina suurempi tai yhtä suuri kuin 0 (nolla).</span><span class="sxs-lookup"><span data-stu-id="f3f58-136">The ATP quantity that is shown is always greater than or equal to 0 (zero).</span></span> <span data-ttu-id="f3f58-137">Jos laskutoimituksen tulos on negatiivinen ATP-määrä (jos aiemmin luvattu määrä on esimerkiksi suurempi kuin käytettävissä oleva määrä), määräksi määritetään automaattisesti 0.</span><span class="sxs-lookup"><span data-stu-id="f3f58-137">If the calculation returns a negative ATP quantity (for example, if the quantity that was previously promised exceeds the available quantity), the quantity is automatically set to 0.</span></span>
