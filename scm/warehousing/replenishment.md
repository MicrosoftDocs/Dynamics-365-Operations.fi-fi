# <a name="replenishment"></a><span data-ttu-id="8db86-103">Täydennys</span><span class="sxs-lookup"><span data-stu-id="8db86-103">Replenishment</span></span>

[!include[banner](../includes/banner.md)]


<span data-ttu-id="8db86-104">Tässä ohjeaiheessa käsitellään täydennysstrategioita, joita voi käyttää varastonhallinnan toimintoja käyttävissä varastoissa.</span><span class="sxs-lookup"><span data-stu-id="8db86-104">This topic describes the replenishment strategies that are available for warehouses which use the functionality that is available in Warehouse management.</span></span> <span data-ttu-id="8db86-105">Tiedot eivät koske Inventoinnin- ja varastonhallinnassa käytettävissä olevaa varastoratkaisua.</span><span class="sxs-lookup"><span data-stu-id="8db86-105">The information doesn't apply to the warehousing solution that is available in Inventory management.</span></span> <span data-ttu-id="8db86-106">Seuraavat täydennysstrategiat ovat käytettävissä:</span><span class="sxs-lookup"><span data-stu-id="8db86-106">The following replenishment strategies are available:</span></span>

-   <span data-ttu-id="8db86-107">**Aallon kysynnän täydennys** – Tämä strategia luo lähteville tilausten tai kuormien täydennystyön, jos varasto ei ole käytettävissä, kun aalto luo työt.</span><span class="sxs-lookup"><span data-stu-id="8db86-107">**Wave demand replenishment** – This strategy creates replenishment work for outbound orders or loads if inventory isn't available when the wave creates work.</span></span> <span data-ttu-id="8db86-108">Täydennystyö voidaan esimerkiksi luoda, jos myyntitilaukseen vaadittava määrä ei ole käytettävissä, kun aalto käsitellään.</span><span class="sxs-lookup"><span data-stu-id="8db86-108">For example, replenishment work can be created if the quantity that is required for a sales order isn't available when a wave is processed.</span></span>
-   <span data-ttu-id="8db86-109">**Vähimmäis- tai enimmäistäydennys** – Tässä strategiassa varastoinnin vähimmäis-ja enimmäisrajoilla määritetään, milloin sijainnit on täydennettävä.</span><span class="sxs-lookup"><span data-stu-id="8db86-109">**Min/Max replenishment** – This strategy uses minimum and maximum stocking limits to determine when locations should be replenished.</span></span> <span data-ttu-id="8db86-110">Nimike ja sijaintiehto määrittävät varaston, joka arvioidaan täydennystä varten.</span><span class="sxs-lookup"><span data-stu-id="8db86-110">The item and location criteria define the inventory that is evaluated for replenishment.</span></span> <span data-ttu-id="8db86-111">Vähimmäis- tai enimmäistäydennykset malli ovat ensisijainen mekanismi optimaalisten tasojen ylläpitämiseksi keräilysijainneissa.</span><span class="sxs-lookup"><span data-stu-id="8db86-111">Min/Max replenishment templates are the primary mechanism for maintaining optimal levels in picking locations.</span></span> <span data-ttu-id="8db86-112">Voit kysynnän täydennyksen avulla varmistaa, että varastoa riittää aallon kysyntään vastaamiseen. Tämä toimii vähimmäis- ja enimmäistäydennysjaksojen välisenä täydennyksenä.</span><span class="sxs-lookup"><span data-stu-id="8db86-112">To ensure that enough pick face inventory is available to meet wave demand, you can use demand replenishment as a supplement between Min/Max replenishment cycles.</span></span>
-   <span data-ttu-id="8db86-113">**Kuorman kysynnän täydennys** – Tässä strategiassa summataan useiden kuormien kysyntä ja luodaan täydennystyö, jota tarvitaan varaston täyttämiseen liittyvissä keräilysijainneissa.</span><span class="sxs-lookup"><span data-stu-id="8db86-113">**Load demand replenishment** – This strategy sums the demand for several loads and creates the replenishment work that is required in order to stock the relevant picking locations.</span></span> <span data-ttu-id="8db86-114">Tämä strategia auttaa varmistamaan, että luodut kuormat voidaan kerätä varastosta vapauttamisen jälkeen.</span><span class="sxs-lookup"><span data-stu-id="8db86-114">This strategy helps to ensure that the loads that are created can be picked in the warehouse after they are released.</span></span>

<span data-ttu-id="8db86-115">Täydennystyö luodaan täydennysmallin perusteella kaikissa kolmessa strategiassa.</span><span class="sxs-lookup"><span data-stu-id="8db86-115">All three strategies create replenishment work, based on a replenishment template.</span></span>

## <a name="wave-demand-replenishment"></a><span data-ttu-id="8db86-116">Aallon kysynnän täydennys</span><span class="sxs-lookup"><span data-stu-id="8db86-116">Wave demand replenishment</span></span>

<span data-ttu-id="8db86-117">Aallon kysynnän täydennys luo kysynnän mukaan täydennystyön, jos tuotantotilausten, kanbanien, lähtevien tilausten tai kuormien määrä ei ole käytettävissä, kun aalto luo työn.</span><span class="sxs-lookup"><span data-stu-id="8db86-117">Wave demand replenishment creates replenishment work, based on demand, if the quantity that is required for production orders, kanbans, outbound orders or loads isn't available when a wave creates work.</span></span> <span data-ttu-id="8db86-118">Täydennysmallissa on tietoja nimikkeen ehdoista, mittayksiköstä, kysynnän lisäyksestä ja sijainnista.</span><span class="sxs-lookup"><span data-stu-id="8db86-118">The replenishment template contains information about the item criteria, the unit of measure, the demand increment, and the location.</span></span> 

<span data-ttu-id="8db86-119">Sijaintidirektiiveillä määritetään, mitä sijaintia on täydennettävä.</span><span class="sxs-lookup"><span data-stu-id="8db86-119">Location directives are used to determine which location should be replenished.</span></span> <span data-ttu-id="8db86-120">Nämä sijaintidirektiivit linkitetään täydennysmalliin **Direktiivikoodi**-kentässä.</span><span class="sxs-lookup"><span data-stu-id="8db86-120">You link these location directives to the replenishment template by using the **Directive code** field.</span></span> <span data-ttu-id="8db86-121">Jos **Direktiivikoodi**-kenttää ei ole määritetty, käytettävä suuntadirektiivi määritetään kyselyillä.</span><span class="sxs-lookup"><span data-stu-id="8db86-121">If the **Directive code** field isn't set, queries are used to determine which location directive should be used.</span></span> <span data-ttu-id="8db86-122">Huomaa, että jos direktiivikoodia ei ole määritetty täydennysmallissa ja sijaintidirektiivillä on direktiivikoodi, sijaintidirektiivi ohitetaan, vaikka sijaintidirektiivikysely olisi oikein.</span><span class="sxs-lookup"><span data-stu-id="8db86-122">Note that if a directive code isn't specified in the replenishment template, and the location directive has a directive code, the location directive will be ignored, even if the query on the location directive is correct.</span></span> <span data-ttu-id="8db86-123">Keräilysijaintidirektiivillä määritetään, mistä varaston täydennys hankitaan.</span><span class="sxs-lookup"><span data-stu-id="8db86-123">Pick location directives are used to determine where to get inventory for the replenishment.</span></span> 

<span data-ttu-id="8db86-124">Mallin luonnin lisäksi voit määrittää joitakin täydennysasetuksia aaltomallissa.</span><span class="sxs-lookup"><span data-stu-id="8db86-124">In addition to creating a template, you must specify some replenishment settings in the wave template.</span></span> <span data-ttu-id="8db86-125">Aaltomallissa on oltava täydennysaaltovaihe, joka suoritetaan vain, jos nimikkeen kohdistus ei onnistunut.</span><span class="sxs-lookup"><span data-stu-id="8db86-125">The wave template should contain a wave step for replenishment that is run only if allocation of an item isn't successful.</span></span> <span data-ttu-id="8db86-126">Täydennysaaltovaihe käyttää aaltovaihekoodia käytettävän täydennysmallin määrittämiseen.</span><span class="sxs-lookup"><span data-stu-id="8db86-126">This replenishment wave step uses a wave step code to determine which replenishment template should be used.</span></span> <span data-ttu-id="8db86-127">Täydennysaaltovaiheen lisäksi on varmistettava, että aaltomallin **Menetelmät**-kohdassa on valittu **Täydennä**.</span><span class="sxs-lookup"><span data-stu-id="8db86-127">In addition to having a wave step for replenishment, you need to make sure that **Replenish** is selected in the **Methods** section of the wave template.</span></span> 

<span data-ttu-id="8db86-128">**Täydennysmalli**-sivulla on **Salli aallon kysynnän käyttää varaamattomia määriä** -valintaruutu.</span><span class="sxs-lookup"><span data-stu-id="8db86-128">The **Replenishment template** page includes an **Allow wave demand to use unreserved quantities** check box.</span></span> <span data-ttu-id="8db86-129">Valitse tämä valintaruutu, jos haluat, että kysynnän täydennys vähentää varaamattomia määriä valitusta täydennysmallista luodusta työstä.</span><span class="sxs-lookup"><span data-stu-id="8db86-129">You should select this check box if you want to allow demand replenishment to deduct unreserved quantities from work that is generated from the selected replenishment template.</span></span> <span data-ttu-id="8db86-130">Tämä valintaruutu on valittava jokaisessa aiemmin luodussa täydennysmallissa, jotta kysynnän täydennysmallit voisivat käyttää tätä logiikkaa.</span><span class="sxs-lookup"><span data-stu-id="8db86-130">To enable demand replenishment templates to use this logic, select this check box for every existing replenishment template.</span></span> <span data-ttu-id="8db86-131">Kun kysynnän täydennys käynnistyy varastossa, se vähentää kysyntää nykyiseltä täydennystyöltä, jolla on varaamattomia määriä, jos työn alkuperä on täydennysmalleissa, joissa **Salli aallon kysynnän käyttää varaamattomia määriä** -valintaruutu on valittuna.</span><span class="sxs-lookup"><span data-stu-id="8db86-131">When demand replenishment is triggered in the warehouse, it will deduct the demand from existing replenishment work that has unreserved quantities, if the work originates from replenishment templates where the **Allow wave demand to use unreserved quantities** check box is selected.</span></span>


<span data-ttu-id="8db86-132">Kysynnän täydennystä tuetaan myyntitilauksissa, siirtotilauksissa, tuotantotilauksissa ja kanbaneissa.</span><span class="sxs-lookup"><span data-stu-id="8db86-132">Demand replenishment is supported for sales orders, transfer orders, production orders, and kanbans.</span></span> 

## <a name="minmax-replenishment"></a><span data-ttu-id="8db86-133">Vähimmäis- tai enimmäistäydennys</span><span class="sxs-lookup"><span data-stu-id="8db86-133">Min/Max replenishment</span></span>
<span data-ttu-id="8db86-134">Vähimmäis- tai enimmäistäydennyksessä varastoa täydennetään niin, että on määritettyjen vähimmäis- ja enimmäisrajojen välillä.</span><span class="sxs-lookup"><span data-stu-id="8db86-134">In Min/Max replenishment, stock is replenished so that it's between the minimum and maximum limits that have been set.</span></span> <span data-ttu-id="8db86-135">Yleensä tämä prosessi tapahtuu kerran päivässä, jotta voidaan varmistaa, että kaikki keräilysijainnit on täytetty enimmäistasolle ennen keräilyn aloittamista.</span><span class="sxs-lookup"><span data-stu-id="8db86-135">Typically, this process occurs one time every day to ensure that all picking locations are filled to the maximum level before picking starts.</span></span> 

<span data-ttu-id="8db86-136">Vähimmäis- ja enimmäissummat määritetään täydennysmallissa.</span><span class="sxs-lookup"><span data-stu-id="8db86-136">The minimum and maximum amounts are set in a replenishment template.</span></span> <span data-ttu-id="8db86-137">Monet muut mallin asetukset muistuttavat aallon kysynnän täydennyksessä käytettäviä malleja.</span><span class="sxs-lookup"><span data-stu-id="8db86-137">Many of the other settings in the template resemble the settings in templates that are used in Wave demand replenishment.</span></span> <span data-ttu-id="8db86-138">Mallissa on oltava yksi rivi kullekin nimikkeelle ja sijainnille.</span><span class="sxs-lookup"><span data-stu-id="8db86-138">The template should contain one line for each item and location.</span></span> <span data-ttu-id="8db86-139">Kun täydennys suoritetaan käyttämällä erätyötä, Finance and Operations tarkistaa täydennyksen tarpeen siinä järjestyksessä, johon rivit on järjestetty.</span><span class="sxs-lookup"><span data-stu-id="8db86-139">When you run replenishment by using the batch job, Finance and Operations evaluates whether replenishment is required, in the sequence that the lines are organized in.</span></span> 

<span data-ttu-id="8db86-140">Huomaa, vähimmäis- ja enimmäistäydennysstrategia ei voi täydentää tyhjää sijaintia, ellei sijaintia ole määritetty nimikkeen kiinteästi sijainniksi.</span><span class="sxs-lookup"><span data-stu-id="8db86-140">Note that the Min/Max replenishment strategy can't replenish an empty location unless the location is set as the fixed location for the item.</span></span> <span data-ttu-id="8db86-141">Jos täydennettävä sijainti ei ole kiinteä sijainti, täydennettävää nimikettä ei voi määrittää.</span><span class="sxs-lookup"><span data-stu-id="8db86-141">If the location that must be replenished isn't a fixed location, it's not possible to determine which item to replenish.</span></span> <span data-ttu-id="8db86-142">Niinpä ennen täydennystä tarvitaan ainakin varastosaldo.</span><span class="sxs-lookup"><span data-stu-id="8db86-142">Therefore, at least some on-hand quantity is required before replenishment occurs.</span></span>
